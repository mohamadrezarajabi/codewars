<h2><a href=https://www.codewars.com/kata/6a1f03fac18f58f98ad9d21a/train/python target="_blank">Password entropy</a></h2><h3>7 kyu</h3><p>We all know that short passwords are not very secure, particularly ones based on names or dictionary words. It's usually a good idea to use a mixture of uppercase and lowercase characters, digits and special characters, although some people favour a password simply composed of several dictionary words: <img alt="https://imgs.xkcd.com/comics/password_strength.png" src="https://imgs.xkcd.com/comics/password_strength.png"></p><p>But how do we measure password security? A simple approach is to calculate the password's entropy <code>E</code>, measured in bits, using the formula:</p><pre><code>E = L * log2(R)</code></pre><p>where:</p><ul><li><code>R</code> is the of the size of the pool of characters from which we build the password</li><li><code>L</code> is the number of characters in the password.</li></ul><p>In this kata passwords are always composed of one or more printable ASCII characters, excluding whitespace. That means:</p><ul><li>26 lowercase alphabetic characters <code>a..z</code></li><li>26 uppercase alphabetic characters <code>A..Z</code></li><li>10 digits <code>0..9</code></li><li>32 special characters <code>!"#$%&amp;'()*+,-./:;&lt;=&gt;?@[\]^_`{|}~</code></li></ul><p>These are the four pools of characters used in the entropy calculation</p><p>For example, the password <code>hello</code> is composed entirely of lowercase alphabetic characters, so the pool size is <code>26</code> and the password length is <code>5</code>. This gives:</p><pre><code>E = 5 * log2(26) ≈ 23.50 bits</code></pre><p>As another example, the password <code>Tr0ub4dor&amp;3</code> uses characters from all four pools, so the combined pool size is <code>94</code> and the password length is <code>11</code>. This gives:</p><pre><code>E = 11 * log2(94) ≈ 72.10 bits</code></pre><h2 id="task">Task</h2><p>Write a function <code>entropy</code> which takes as its input a valid password and returns the entropy of the password, calculated using the formula above.</p>
